This tool is used for submitting private stress run requests. You should have an enlistment on your machine, and launch the tool from a Razzel window. When you submit the form, the specified build flavor(s) will be built on your machine, and then copied to a network share. You will be notified of the results. You may also track status at Stress Run Status Monitor.
Build Flavors
The build flavor(s) you want to generate and run stress on. The default is x86chk, x86ret, amd64chk, and amd64ret.
Test Mixes
Specific stress test mix(es) you want to run. If not specified, they will be selected at random.
GFlags Options
See GFlags Help for more information.
MDAs Options
Managed debugging assistants (MDAs) you want to enable during stress runs. Some MDAs are implicitly activated when a managed or unmanaged executable is started under a debugger. See MDAs User's Guide for more information.
Environment Variables
And additional environment variables you want to specify, separated by semi-colon. For example, "COMPlus_GCStress=3;COMPlus_HeapVerify=1".
